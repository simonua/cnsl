## Evidence-Based Exclusions

Current evidence does not support a framework or bundler migration, generic route-controller abstraction, universal script loader, annual-document split, per-card requests, relationship payload, persistent model cache, runtime index, pre-expanded dates, `content-visibility`, or virtualization. Collections remain small, annual requests are shared, installed navigation is controlled and transfer-free, and the active payloads compress well.

Do not add these mechanisms unless a current consumer or repeatable measurement establishes a specific constraint and the proposed implementation names the code or delivery commitment it replaces.

## Active Roadmap

There is no active refactoring roadmap. Start a new evidence-led review when one of these triggers occurs:

- A current producer cannot satisfy an active annual schema or a runtime consumer requires a second representation.
- Comparable route measurements identify a stable phase or long task that materially delays useful content.
- A dependency, global, adapter, fixture, or compatibility path loses its last verified current consumer.
- Build, PWA, accessibility, security, or focused behavioral verification reports a reproducible defect.

## Guardrails

- Do not modify `src/assets/data/` during general refactoring. Seasonal sources require live official-source review, recorded evidence, baseline refresh, and human approval through the annual-data workflow.
- Never edit `out/`; it is generated by `pnpm run build`.
- Preserve `getDataManager()`, one shared pending request and manager collection per annual domain, and source ownership across pools, teams, and meets.
- Retire obsolete definitions, callers, fallbacks, aliases, fixtures, tests, types, dependencies, registrations, documentation, build rules, and cache references together.
- Preserve the PostHTML static-site architecture, native DOM APIs, classic browser-script boundary, CSP and rendering trust boundaries, analytics privacy limits, WCAG 2.0 AA behavior, and current offline commitments.
- Keep performance budgets advisory until repeatable local and approved delivered-site samples establish stable variance. Do not navigate an automated browser to production.
- Run only change-scoped unit and browser tests for implementation work. Keep changed delivered modules at 100% focused coverage when coverage is collected, and run required lint, build, data, and PWA gates according to the affected boundary.
